# Checklist Phase 1 - Nền Tảng Repo Và Quy Ước

## Tài Liệu

- [ ] Rà soát `README.md`.
  - [ ] Mô tả đúng trạng thái dự án.
  - [ ] Có keyword phù hợp cho GitHub.
  - [ ] Có link tới README tiếng Việt.
- [ ] Rà soát `README.vi.md`.
  - [ ] Toàn bộ tiếng Việt có dấu chuẩn.
  - [ ] Không dùng tiếng Việt không dấu trừ identifier, path, command.
- [ ] Rà soát `ROADMAP.md`.
- [ ] Rà soát `CONTRIBUTING.md`.
- [ ] Rà soát `SECURITY.md`.
- [ ] Rà soát `docs/README.md`.

## Quy Ước Agent

- [ ] Cập nhật `AGENTS.md` khi quy tắc triển khai thay đổi.
- [ ] Cập nhật `.codex/project-context.md` khi hướng sản phẩm thay đổi.
- [ ] Cập nhật `.codex/implementation-rules.md` khi đổi quy tắc kỹ thuật.
- [ ] Thêm task phase mới trong `.codex/tasks/` khi bắt đầu phase mới.

## GitHub

- [ ] Kiểm tra repo đang private.
- [ ] Kiểm tra remote `origin`.
- [ ] Kiểm tra CI chạy trên GitHub.
- [ ] Kiểm tra issue templates.
- [ ] Kiểm tra PR template.
- [ ] Tạo labels cơ bản.
  - [ ] `bug`
  - [ ] `enhancement`
  - [ ] `documentation`
  - [ ] `security`
  - [ ] `backend`
  - [ ] `frontend`
  - [ ] `ai`

## Quyết Định Cần Chốt

- [ ] License.
- [ ] Package manager.
- [ ] Rust workspace layout.
- [ ] API contract strategy.
- [ ] Database migration strategy.
- [ ] Secret encryption strategy.
